Règle tactile pédagogique

Modélisation paramétrique · Impression 3D · Solo

Rôle · Conception complète
Année · 2024
Statut · Terminé

Contexte

Concevoir un outil pédagogique tangible pour les enfants malvoyants : une règle dont chaque graduation, chaque repère est perceptible au toucher. Le défi n'est pas graphique mais physique - dimensions, épaisseurs, espacement des reliefs doivent être pensés pour les doigts, pas pour les yeux.

L'approche paramétrique permet à n'importe quel enseignant ou orthoptiste de modifier les dimensions selon les besoins spécifiques d'un enfant, sans toucher au code de modélisation.

SolidPython2 OpenSCAD Impression 3D Accessibilité CAO

Problème & solution

Problème

Les règles standard sont conçues pour la vision. Les graduations sont imprimées, pas en relief. Les chiffres sont visuels. Pour un enfant malvoyant, ces outils sont inutilisables sans adaptation spécifique et coûteuse.

Solution

Modélisation 100% paramétrique en SolidPython2 générant du code OpenSCAD. Chaque dimension est une variable : longueur totale, espacement des graduations, hauteur des reliefs, taille des chiffres en relief. Une ligne de code suffit à adapter la règle à un nouveau besoin. Export STL direct pour impression 3D.

Aperçu de la logique paramétrique

Pas de schéma interactif pour ce projet : la modélisation se lit directement dans le code. Voici une version simplifiée de la logique paramétrique utilisée.

# Exemple simplifié - paramètres modifiables
LONGUEUR     = 200  # mm
NB_GRADUATIONS = 20
HAUTEUR_RELIEF = 0.8  # mm perceptible au toucher

def graduation(position, hauteur):
    return cube([2, hauteur * 3, HAUTEUR_RELIEF])
           .translate([position, 0, 0])

Démarche technique

01
Analyse des besoins
Étude des contraintes tactiles : quelle hauteur de relief est perceptible du bout des doigts ? Quel espacement entre graduations évite la confusion ? Recherche sur les standards d'accessibilité physique (hauteur relief minimum 0.5mm, largeur graduation recommandée).
02
Modélisation paramétrique
Écriture du modèle en SolidPython2 : chaque élément (corps de règle, graduation courte, graduation longue, chiffre en relief) est une fonction Python qui prend des paramètres. OpenSCAD génère le solide 3D à partir du code. Itérations rapides : modifier un paramètre = régénérer instantanément.
03
Impression et validation
Export en STL, impression sur imprimante FDM. Vérification tactile des reliefs : hauteur suffisante, arêtes non coupantes, stabilité de la pièce. Ajustements des paramètres selon le retour physique.

Ce que ce projet démontre

Ce projet prouve que l'accessibilité ne se limite pas au numérique. Appliquer une logique de conception inclusive à un objet physique - penser pour les doigts plutôt que pour les yeux - demande la même rigueur méthodologique qu'un audit WCAG.

Il démontre aussi ma maîtrise de la modélisation paramétrique (SolidPython2/OpenSCAD) et ma capacité à traduire un besoin utilisateur concret en contraintes techniques précises.
